1)Что такое метод?
Метод — это функция, которая определена внутри класса и предназначена для работы с его объектами.  

2)Что такое "функция-член"?
Функция-член — это еще одно название метода.
 
3)Что такое член класса?
Член класса — это переменная или метод, объявленная внутри класса.   
 
4)Чем отличается instance функция (функция-экземпляр) от статической?
Функция-экземпляр требует создания объекта класса для вызова и может обращаться к членам экземпляра (переменным) и другим методам этого экземпляра.
Статическая функция может быть вызвана без создания объекта класса и не имеет доступа к членам экземпляра. Она может обращаться только к статическим членам класса.
 
5)Какой синтаксис вызова методов?
Синтаксис вызова метода экземпляра:
 object.method(parameter); // object - экземпляр класса, а method - функция-член
Синтаксис вызова статического метода:
ClassName::staticMethod(parameter); // ClassName - имя класса

6)Как только объявить метод, а определить его отдельно?
class MyClass {
public:
   static void myMethod(); // Объявление метода
};

// Определение метода вне класса
void MyClass::myMethod() {
     std::cout << "Hello" << std::endl; 
}
 
7)Возможно ли определить метод внутри класса, который будет статический для линкера (определен только для текущего compilation unit-a)?
Один конкретный — нельзя. Можно все методы в классе, приписав static к определению класса. Некоторым образом это можно обойти, используя вложенный класс. 
 
8)Обязателен ли оператор new чтобы создать класс (структуру с методами)?
Нет, оператор new не обязателен для создания объектов класса. Вы можете создать объекты на стеке, что не требует использования new:
 MyClass obj;  
Использование new создает объект в динамической памяти:
MyClass* obj = new MyClass(); 

9)Влияет ли количество методов класса на то сколько байт будет занимать объект типа этого класса?
Нет, количество методов не влияет на размер объекта. Размер объекта зависит от его   переменных, а не от методов, которые могут быть в классе. Методы не занимают место в памяти объекта.
 
10)Занимает ли объект с одним методом больше памяти, чем объект без методов?
Нет, объект с одним методом не занимает больше памяти, чем объект без методов. 

11)В чем функциональное преимущество использование методов вместо обычных функций, принимающих параметр, подобный параметру this, не скрытно, первым параметром.
 
Методы имеют преимущество в плане структурирования кода(читаемость).
